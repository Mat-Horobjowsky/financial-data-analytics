Coding Standards
Keep modules small and focused.
Use clear function names and type hints.
Prefer simple pandas logic.
Raise clear custom errors where helpful.
Avoid overengineering.
Avoid hidden business logic.
Make failures easy to understand.
Keep every output deterministic and testable.

The user prefers concise, modular, reusable systems that can scale iteratively. Do not restart from scratch unnecessarily. Prioritize working MVPs that can become production-grade over large speculative architectures.
